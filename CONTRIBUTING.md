# Як долучитися

Дякуємо за інтерес до бібліотеки BFU Electronics.

## Головне правило

**Не видумуй розміри.** Кожна координата, крок падів і габарит мають
походити з офіційного даташита виробника або перевіреної бібліотеки
(Espressif, KiCad, LCSC/JLCPCB зі звіркою по кресленню).

Футпринт із неперевіреними розмірами — це перевипуск плати й викинуті гроші.
Краще залишити категорію порожньою.

## Процес

1. Створи issue за шаблоном «Новий компонент» — щоб не робити ту саму роботу двічі.
2. Форкни репозиторій, створи гілку:
   ```bash
   git checkout -b add/st7789v
   ```
3. Додай футпринт за інструкцією [docs/ADDING.md](docs/ADDING.md).
4. Поклади PDF даташита в `datasheets/`.
5. Перевір:
   ```bash
   python scripts/validate.py
   ```
6. Відкрий PR, заповни чекліст.

## Стиль комітів

```
add(BFU_Display): ST7789V — 240x320 TFT, FPC 8-pin
fix(BFU_MCU): ESP32-S3-WROOM-1 courtyard 0.25mm clearance
docs: інструкція підключення для KiCad 9
```

Префікси: `add`, `fix`, `docs`, `chore`.
У дужках — категорія бібліотеки, якої стосується зміна.

## Що приймається

- Футпринти компонентів, які реально використовуються в проєктах BFU
- Виправлення розбіжностей із даташитом
- Покращення документації

## Що не приймається

- Футпринти без вказаного джерела розмірів
- Копії з інших бібліотек без звірки з даташитом
- Зміни, що ламають `scripts/validate.py`

## Мова

Документація, issue й коміти — українською.
Імена файлів, партномери, поля всередині футпринтів — англійською.
