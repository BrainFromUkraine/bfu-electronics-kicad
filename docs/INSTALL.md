# Підключення бібліотеки в KiCad 9

## Крок 1 — клонувати репозиторій

```bash
cd C:\Users\oleks\Documents\KiCad
git clone https://github.com/BrainFromUkraine/bfu-electronics-kicad.git
```

Отримаєш:
`C:\Users\oleks\Documents\KiCad\bfu-electronics-kicad\BFU_Electronics\`

## Крок 2 — змінна шляху (необов'язково, але зручно)

**Налаштування → Налаштувати шляхи** → кнопка `+`

| Ім'я | Шлях |
|---|---|
| `BFU_LIB` | `C:\Users\oleks\Documents\KiCad\bfu-electronics-kicad\BFU_Electronics` |

Після натискання OK змінна з'явиться внизу вікна бібліотек
у списку «Доступні заміни шляху».

## Крок 3 — додати бібліотеки

**Налаштування → Керування бібліотеками посадкових місць** →
вкладка **Глобальні**

Найпростіше — кнопка з іконкою **теки**:

1. Зайди **всередину** `BFU_Electronics`
2. Виділи потрібні теки `.pretty` (кілька — через Ctrl)
3. **Вибрати теку** → KiCad сам підставить шлях і псевдонім

Або вручну через `+`:

| Псевдонім | Шлях | Формат |
|---|---|---|
| `BFU_RF` | `${BFU_LIB}/BFU_RF.pretty` | KiCad |
| `BFU_MCU` | `${BFU_LIB}/BFU_MCU.pretty` | KiCad |

> **Важливо:** бібліотека — це тека з суфіксом `.pretty`, а не батьківська
> тека `BFU_Electronics` і не окремий файл `.kicad_mod`.

## Крок 4 — перевірити

Відкрий **Редактор посадкових місць** — у дереві зліва мають бути
`BFU_RF` і `BFU_MCU` з компонентами всередині.

## Оновлення

```bash
cd C:\Users\oleks\Documents\KiCad\bfu-electronics-kicad
git pull
```

Бібліотеки в KiCad перепідключати не треба — шляхи ті самі.

## Часті помилки

| Симптом | Причина |
|---|---|
| «формат не підтримується» | вибрано файл `.kicad_mod` замість теки `.pretty` |
| бібліотека порожня | додано батьківську теку `BFU_Electronics` |
| футпринти зникли після оновлення KiCad | бібліотека лежала в `C:\Program Files\KiCad` |
