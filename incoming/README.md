# incoming — тимчасова тека для нових футпринтів

Файли з чату зберігаються СЮДИ, а не напряму в .pretty.
Причина: браузер може перезаписати наявний файл, якщо ім'я збігається.
Так уже двічі постраждав MAX98357A.kicad_mod.

Порядок роботи:
1. зберегти .kicad_mod у incoming/
2. агент перевіряє, що внутрішнє ім'я (footprint "...") збігається
   з іменем файлу, і що файл із таким іменем ще не існує в цільовій .pretty
3. агент переносить файл у потрібну категорію
4. python scripts/validate.py
5. коміт

Тека має лишатися порожньою між сесіями.
